Product Requirements Document (PRD): GoQRCard.com (v1.1 Updated)
1. 프로젝트 개요 (Overview)
서비스명: GoQRCard.com (by BrandFirst.ai)
서비스 정의: 기존 종이 명함의 가치(디자인, 브랜드 아이덴티티)는 유지하면서, QR 코드를 통해 디지털 행동(전화, 메시지, 재접속, 관리)을 추가하는 접점 중심 디지털 명함 플랫폼.
핵심 슬로건:
👉 "쓰던 명함 그대로, 기능만 스마트하게."
👉 "저장하지 않아도, 바로 연결되는 명함"
2. 배경 및 목적 (Background & Objectives)
2.1 문제 정의
종이 명함은 전달 이후 활용 여부 및 접점 데이터를 알 수 없음.
기존 디지털 명함은 제작 과정이 복잡하고, 주소록 저장 이후 서비스 재방문이 발생하지 않음.
디지털 명함 서비스는 대부분 “1회 사용 후 종료”되는 구조로 리텐션이 낮음.
2.2 해결 방안
명함 사진 기반 10초 생성 구조로 진입 장벽 제거.
명함을 “저장 대상”이 아닌 즉시 행동 가능한 인터페이스로 전환.
Viewer를 자연스럽게 App 경험 → Owner 전환으로 유도하는 UX 설계.
주소록이 아닌 **웹 기반 접근 + 홈화면 추가(PWA)**로 리텐션 확보.
2.3 목표
QR 스캔 → 행동(전화/메시지) 전환 극대화
“저장 없이 사용” 패턴 확립
Viewer → Owner 전환률 확보
웹앱 기반 반복 사용 구조 구축
3. 핵심 타겟 (Target Audience)
Owner (생성자):
영업, BD, 창업자, 프리랜서
오프라인 접점이 많은 사용자
Viewer (수신자):
명함을 받고 즉시 연락하거나 확인하려는 사용자
빠른 행동을 선호하는 모바일 사용자
4. 사용자 흐름 (User Flow)
4.1 Owner: 명함 생성 및 관리 (No-Login First)
Entry → 랜딩 페이지 → [무료로 바로 만들기]
Input → 명함 이미지 업로드 + 이름/전화/이메일 입력
Output → QR + URL 즉시 생성
Distribution → QR 공유 (종이/모바일)
Conversion → “수정 / 통계 / 관리” 시점에서 로그인 유도
4.2 Viewer: 명함 열람 및 행동
Scan → QR 스캔
View → 명함 이미지 중심 화면
Action → 전화 / 메시지 / 이메일 (즉시 실행)
Optional → 저장 / 홈화면 추가
Transition → “명함 관리 시작하기” → App 모드 진입
4.3 Viewer → Owner 전환 흐름
QR → 명함 경험 → 행동 → 관심 → 명함 관리 시작 → 생성

6. UI/UX 설계 원칙 (추가)
6.1 핵심 원칙
Viewer UX는 5초 안에 끝나야 한다
“저장”보다 “즉시 행동”을 우선한다
강제 가입 금지 (Progressive Onboarding)
앱 경험은 선택적으로 노출한다
6.2 인터랙션 설계
명함 클릭 → 앞/뒤 전환
오른쪽 스와이프 → 명함 생성 유도
버튼 클릭 후 → 전환 화면 노출
6.3 전환 UX
이 명함, 꽤 편하지 않았나요?
👉 나도 이런 명함 만들기
6.4 리텐션 UX
홈화면 추가 유도 (행동 이후)
최근 접점 자동 생성
반복 접근 경로 제공
6.5 모드 분리
모드	특징
